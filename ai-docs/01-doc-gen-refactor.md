# Docs Refactoring Plan

## Problem

- Manual Hugo pages duplicate generated content (stale immediately)
- `hugoExamplesTmpl` just re-renders `hack/dev-samples.yaml` — `make generate` could produce `config/samples/` directly from CRD defaults
- `docGenDiagramTmpl` describes the generator's own internals — stale the moment any template/source changes
- Several manual pages overlap each other (monitoring.md ≈ observability.md, crds.md ≈ _generated_crds.md, getting-started.md ≈ install.md + usage.md)

## Audit Summary

### Delete (stale/redundant)

| File | Reason |
|------|--------|
| `docs/content/docs/crds.md` | Hand-written CRD ref, duplicates `_generated_crds.md` |
| `docs/content/docs/getting-started.md` | Merge of install.md + usage.md, both exist separately |
| `docs/content/docs/observability.md` | Identical to monitoring.md |
| `docs/content/docs/kamera.md` | Internal evaluation note, not user-facing |
| `docs/doc-generation.md` | Generated by `docGenDiagramTmpl`, stale immediately |
| `docs/content/docs/_generated_examples.md` | Generated by `hugoExamplesTmpl`, redundant (see below) |

### Remove from generator

| Template | Reason |
|----------|--------|
| `docGenDiagramTmpl` | Self-referential diagram of the generator — stale on any change |
| `hugoExamplesTmpl` | Just wraps `dev-samples.yaml` in Hugo frontmatter. Replace with `config/samples/` generation from CRD types via `make generate` |

### Keep (generated, valuable)

| Template | Output | Why |
|----------|--------|-----|
| `hugoCRDsTmpl` | `reference/_generated_crds.md` | CRD field truth from AST |
| `hugoErrorsTmpl` | `reference/_generated_errors.md` | Error reasons from source |
| `hugoMetricsTmpl` | `reference/_generated_metrics.md` | Metrics from registration code |
| `hugoArchTmpl` | `reference/_generated_architecture.md` | Package graph + sequence diagrams |

### Keep (manual, valuable)

| File | Content |
|------|---------|
| `install.md` | Helm install + verify |
| `usage.md` | Narrative examples per CRD |
| `discovery.md` | How discovery works (unique flow) |
| `monitoring.md` | Metrics + events + dashboards |
| `for-ai-agents.md` | Explains llms.txt endpoints |
| `developing/*` | Architecture, conventions, testing, debugging, extending, releasing |
| `proof-of-operation.md` | Annotated output from prove-operator.sh |

## Plan

### Phase 1: Remove stale generated outputs

1. Delete `docGenDiagramTmpl` and `hugoExamplesTmpl` from `templates.go`
2. Remove their entries from `outputTargets()` in `config.go`
3. Delete `docs/doc-generation.md` and `docs/content/docs/_generated_examples.md`

### Phase 2: Remove stale manual pages

1. Delete `docs/content/docs/crds.md` (replaced by `_generated_crds.md`)
2. Delete `docs/content/docs/getting-started.md` (split already exists as install.md + usage.md)
3. Delete `docs/content/docs/observability.md` (duplicate of monitoring.md)
4. Delete `docs/content/docs/kamera.md` (internal note, not docs)

### Phase 3: Generate `config/samples/` from CRD types

Instead of `hugoExamplesTmpl`, make `make generate` produce minimal valid YAML samples in `config/samples/` by reading required fields + defaults from the CRD types. This is what kubebuilder already does — we just keep `hack/dev-samples.yaml` for comprehensive/advanced examples.

### Phase 4: Fix Hugo navigation

Update `_index.md` files to remove dead links and reflect the trimmed structure:

```
docs/
  install.md          — Helm install
  usage.md            — Examples per CRD (narrative)
  discovery.md        — DiscoveryPolicy guide
  monitoring.md       — Metrics, events, dashboards
  for-ai-agents.md    — AI endpoint docs
  developing/         — Dev guides (architecture, testing, etc.)
  reference/          — Generated (CRDs, errors, metrics, architecture)
  proof-of-operation.md
```

## Non-goals

- Don't rewrite the developing/ guides (they're stable and unique)
- Don't move generated reference pages (URLs have aliases)
- Don't generate narrative docs — only schema/catalog docs benefit from generation
